# Rúbrica de Evaluación – Proyecto Django (Nivel Inicial)

**Materia:** Programación 2  
**Carrera:** Tecnicatura Universitaria en Programación – UTN FRLP  
**Modalidad:** Trabajo grupal  
**Evaluación:** Final del proyecto integrador (entrega + exposición)  
**Puntaje total:** 100 puntos

---

## 1. Presentación y documentación (15 puntos)

| Nivel            | Descripción                                                                                                                                                                                                 | Puntos |
| :--------------- | :---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | :----: |
| **Excelente**    | Presenta con claridad los objetivos, funciones principales y estructura del proyecto. Incluye `README` completo (instalación, ejecución, roles del equipo, dependencias). Uso adecuado de lenguaje técnico. | **15** |
| **Bueno**        | Presenta objetivos y funciones, pero con documentación parcial o incompleta. Falta algún aspecto menor del `README` o explicación oral.                                                                     | **10** |
| **Básico**       | Se presenta el proyecto sin un orden claro, faltan detalles o documentación mínima.                                                                                                                         | **5**  |
| **Insuficiente** | No se presenta documentación ni se logra explicar correctamente el funcionamiento del proyecto.                                                                                                             | **0**  |

## 2. Trabajo en equipo y roles (15 puntos)

| Nivel            | Descripción                                                                                                                                | Puntos |
| :--------------- | :----------------------------------------------------------------------------------------------------------------------------------------- | :----: |
| **Excelente**    | Evidente coordinación entre miembros. Roles definidos (backend, frontend, documentación, testing). Participación equitativa en exposición. | **15** |
| **Bueno**        | Colaboración aceptable. Algunos miembros participaron más que otros o los roles no fueron totalmente claros.                               | **10** |
| **Básico**       | Colaboración limitada. Un integrante llevó el peso principal del trabajo.                                                                  | **5**  |
| **Insuficiente** | No se evidencia trabajo grupal o el proyecto fue realizado individualmente.                                                                | **0**  |

## 3. Uso de Model (20 puntos)

| Nivel            | Descripción                                                                                                                                                                     | Puntos |
| :--------------- | :------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ | :----: |
| **Excelente**    | Los modelos representan correctamente las entidades del dominio. Relaciones bien definidas (OneToMany, ManyToMany). Uso adecuado de tipos de campo, `__str__()`, y migraciones. | **20** |
| **Bueno**        | Modelos funcionales, con pequeñas deficiencias en tipos de campo o relaciones.                                                                                                  | **15** |
| **Básico**       | Modelos muy simples o con errores de estructura. Relaciones mal definidas o ausentes.                                                                                           | **10** |
| **Insuficiente** | No hay uso coherente de modelos o están vacíos.                                                                                                                                 | **0**  |

## 4. Uso de Views y lógica (20 puntos)

| Nivel            | Descripción                                                                                                                            | Puntos |
| :--------------- | :------------------------------------------------------------------------------------------------------------------------------------- | :----: |
| **Excelente**    | Las vistas están correctamente estructuradas (funcionales o basadas en clases). Lógica clara, separación entre control y presentación. | **20** |
| **Bueno**        | Las vistas funcionan, pero hay código mezclado o duplicado.                                                                            | **15** |
| **Básico**       | Lógica desorganizada o difícil de seguir.                                                                                              | **10** |
| **Insuficiente** | Las vistas no responden o están vacías.                                                                                                | **0**  |

## 5. Funcionalidad general del sitio (20 puntos)

| Nivel            | Descripción                                                                                                              | Puntos |
| :--------------- | :----------------------------------------------------------------------------------------------------------------------- | :----: |
| **Excelente**    | El sitio es completamente funcional: alta, baja, modificación y consulta. No hay errores visibles. URLs y rutas limpias. | **20** |
| **Bueno**        | Funciona la mayoría de las partes, pero hay errores menores.                                                             | **15** |
| **Básico**       | Funciona parcialmente o con errores frecuentes.                                                                          | **10** |
| **Insuficiente** | El sitio no ejecuta las funciones principales.                                                                           | **0**  |

## 6. Diseño web y usabilidad (10 puntos)

| Nivel            | Descripción                                                                                                             | Puntos |
| :--------------- | :---------------------------------------------------------------------------------------------------------------------- | :----: |
| **Excelente**    | Diseño limpio y ordenado. Uso de plantillas (`extends`, `block`, `include`), maquetación responsive básica y coherente. | **10** |
| **Bueno**        | Diseño aceptable, aunque con inconsistencias o sin responsive.                                                          | **7**  |
| **Básico**       | Plantillas básicas, sin estructura o con errores de formato.                                                            | **4**  |
| **Insuficiente** | No hay diseño ni estructura visual.                                                                                     | **0**  |

---

## 7. Bonus (hasta +5 puntos)

| Criterio                                                                | Puntos |
| :---------------------------------------------------------------------- | :----: |
| Uso de panel de administración con modelos registrados y personalizados | **+2** |
| Implementación de autenticación básica (login/logout)                   | **+2** |
| Integración de imágenes, íconos o estilos propios                       | **+1** |
